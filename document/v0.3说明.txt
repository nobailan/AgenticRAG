3.2 界面操作
-----------
1. 浏览器打开 http://127.0.0.1:7860
2. 在底部输入框输入问题，按回车或点击"发送"
3. 观察：
   - 左侧对话区：答案逐字出现
   - 右侧推理轨迹：逐节点显示 Agent 决策过程
   - 展开"检索来源"：查看检索到的文档块
4. 调整 Top-K 滑块（1-20）和 Temperature 滑块（0-1）
   下次提问时生效
5. 点击"清空对话"重置界面

3.3 推理轨迹示例
---------------
界面右侧"推理轨迹"面板会实时显示：

4.1 config.from_dict()
---------------------
from config import config

# 创建覆盖了 top_k 和 temperature 的新配置
custom = config.from_dict({"top_k": 10, "llm_temperature": 0.5})
# 原始 config 不变，custom 是深拷贝

4.2 get_llm_response_stream()
-----------------------------
from llm_client import get_llm_response_stream

# 逐 token 流式输出
for token in get_llm_response_stream("Say hello"):
    print(token, end="", flush=True)

# 带 callback
def on_token(event_type, token):
    print(f"[{event_type}] {token}")

for token in get_llm_response_stream(
    "Say hello",
    system_prompt="Be brief.",
    temperature=0.5,
    callback=on_token,
):
    pass  # tokens already handled by callback

4.3 run_workflow_streaming()
----------------------------
from workflow import run_workflow_streaming

# events 是 generator，每个 event 是 dict
for event in run_workflow_streaming(
    "What is Veracier?",
    config_override={"top_k": 5, "llm_temperature": 0.2},
):
    etype = event["type"]
    if etype == "node":
        print(f"[{event['node']}] {event['message']}")
    elif etype == "token":
        print(event["content"], end="", flush=True)
    elif etype == "done":
        print(f"\nAnswer: {event['final_answer'][:100]}...")
        print(f"Sources: {event['retrieved_sources']}")

五、事件数据结构
========================================================================

run_workflow_streaming 产出的 3 种事件：

5.1 node 事件
{
    "type": "node",
    "node": "classify_intent",        # 节点名
    "message": "意图分类完成: simple",  # 中文用户消息
    "detail": "simple"                 # 详细结果值
}

5.2 token 事件
{
    "type": "token",
    "content": "Veracier",            # 单个 token
    "accumulated": "Veracier Indu..."  # 已累积的完整答案
}

5.3 done 事件
{
    "type": "done",
    "final_answer": "Based on the...", # 完整答案（含引用）
    "retrieved_sources": ["doc_0", ...],  # 使用的 chunk_id 列表
    "retrieved_chunks": [             # chunk 详情（供来源面板）
        {
            "chunk_id": "doc_124",
            "text": "...",            # 前 200 字符
            "source_file": "veracier_sa/xxx.pdf",
            "score": 0.0255
        }
    ],
    "reasoning_trace": [...],         # 完整推理轨迹
    "intent": "simple"
}

========================================================================
八、已知限制
========================================================================

1. Gradio 6.x Chatbot 不支持 type="messages"（标准格式仍可用）
2. 推理轨迹中文在 Windows GBK 终端可能乱码（浏览器中正常显示）
3. 公网链接需网络畅通（share=True 通过 gradio.live 隧道）
4. 120 秒超时未硬编码（Gradio queue 默认无超时）
5. 多用户并发通过 demo.queue() 排队，无独立会话管理
