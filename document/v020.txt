三、使用方法
========================================================================

3.1 环境准备（一次性）
---------------------
# 1. 激活虚拟环境
E:\anaconda3\envs\graph\Scripts\activate

# 2. 安装依赖（如未安装）
pip install -r requirements_v0_1.txt

# 3. 设置 API Key（二选一）
# PowerShell:
$env:DEEPSEEK_API_KEY = "sk-xxxxxxxxxxxxxxxx"
# 或 OpenAI:
$env:OPENAI_API_KEY = "sk-xxxxxxxxxxxxxxxx"

# 4. 设置 LLM 模型（DeepSeek 用户必须）
$env:RAG_LLM_MODEL = "deepseek-v4-pro"

# 5. 设置本地 Embedding 模型路径
$env:RAG_EMBEDDING_MODEL = "E:/agentProject/embedding-model/bge-base-en-v1.5"

# 6. （可选）使用更小的模型以加速
$env:RAG_EMBEDDING_MODEL = "E:/agentProject/embedding-model/bge-small-en-v1.5"
# 注意: bge-small-en-v1.5 是 384 维，需同步设置:
$env:RAG_EMBEDDING_MODEL = "E:/agentProject/embedding-model/bge-small-en-v1.5"
# （data_prepare.py 会自动更新 embedding_dim）


3.2 生成/重建索引
-----------------
# 快速测试（10 个 PDF）
python data_prepare.py --limit 10 --skip-ocr `
    --embedding-model "E:/agentProject/embedding-model/bge-base-en-v1.5"

# 标准索引（200 个 PDF，推荐）
python data_prepare.py --limit 200 --skip-ocr `
    --embedding-model "E:/agentProject/embedding-model/bge-base-en-v1.5"

# 完整索引 + OCR（全部 PDF，含扫描件，耗时长）
python data_prepare.py --limit 500 `
    --embedding-model "E:/agentProject/embedding-model/bge-base-en-v1.5"

# 自定义输出目录
python data_prepare.py --limit 200 --skip-ocr `
    --output-dir ./my_indexes `
    --embedding-model "E:/agentProject/embedding-model/bge-base-en-v1.5"

# 注意：
# - --skip-ocr 跳过 scanned 和 mixed 格式的 PDF（约 25%）
# - 不加 --skip-ocr 会尝试 OCR 处理扫描件（需安装 Tesseract）
# - --embedding-model 接受 HuggingFace 模型名或本地绝对路径


3.3 运行问答
------------
# 单个问题（需要先设置环境变量）
RAG_LLM_MODEL=deepseek-v4-pro `
RAG_EMBEDDING_MODEL="E:/agentProject/embedding-model/bge-base-en-v1.5" `
python main.py "What is Veracier Industries?"

# 交互模式
python main.py --interactive

# 交互模式命令：
#   /help    — 帮助
#   /quit    — 退出
#   /status  — 索引状态（chunks数、LLM配置等）
#   /graph   — 工作流拓扑

# 批量处理
python main.py --file questions.txt

# 输出到文件
python main.py "R&D spending 2022" --output answer.json

# 详细日志
python main.py --interactive --verbose


3.4 运行测试
-----------
# 结构性测试（无需索引）
python test_mvp.py --verbose

# 端到端测试（需要索引 + LLM）
RAG_LLM_MODEL=deepseek-v4-pro `
RAG_EMBEDDING_MODEL="E:/agentProject/embedding-model/bge-base-en-v1.5" `
python test_e2e.py --verbose --limit 5

# 快速 e2e（2 个问题）
RAG_LLM_MODEL=deepseek-v4-pro `
RAG_EMBEDDING_MODEL="E:/agentProject/embedding-model/bge-base-en-v1.5" `
python test_e2e.py --limit 2

四、本地 Embedding 模型参考
========================================================================

路径: E:\agentProject\embedding-model\

┌─────────────────────────┬──────┬───────┬──────────────────────────┐
│ 模型目录                 │ 维度 │ 大小  │ 适用场景                  │
├─────────────────────────┼──────┼───────┼──────────────────────────┤
│ bge-base-en-v1.5        │ 768  │ 438MB │ 英文为主的多语言检索（推荐）│
│ bge-small-en-v1.5       │ 384  │ 133MB │ 英文快速检索（精度略低）    │
│ bge-small-zh-v1.5       │ 512  │ 95MB  │ 中文优化检索                │
│ bert-base-chinese       │ 768  │ 412MB │ 中文基础模型（无指令微调）  │
└─────────────────────────┴──────┴───────┴──────────────────────────┘

切换模型：
  1. 通过命令行参数（仅 data_prepare.py）:
     --embedding-model "E:/agentProject/embedding-model/bge-small-en-v1.5"

  2. 通过环境变量（全局生效，推荐）:
     $env:RAG_EMBEDDING_MODEL = "E:/agentProject/embedding-model/bge-small-en-v1.5"

  3. data_prepare.py 会自动检测模型实际维度并更新 config.embedding_dim，
     无需手动修改配置。

注意：
  - 切换模型后必须重新运行 data_prepare.py 重建索引
  - FAISS 索引维度必须与 embedding 模型维度匹配
  - 当前 faiss.index 是 768 维（bge-base-en-v1.5）

========================================================================
八、已知限制与后续方向
========================================================================

当前限制:
  1. 仅处理了 200 个 PDF 中的 150 个可搜索 PDF（50 个扫描件跳过）
     完整数据集有 1022 个唯一 PDF
  2. OCR 扫描件处理未启用（需安装 Tesseract + poppler）
  3. bge-base-en-v1.5 是英文优化模型，数据集含法语/德语文档
  4. 未使用 Cross-encoder 重排序
  5. 未实现多轮对话

后续改进方向（v0.3 候选）:
  - 启用 OCR 处理扫描件
  - 添加 Cross-encoder 重排序提升精度
  - 多语言 embedding 模型（如 paraphrase-multilingual）
  - 语义缓存减少 LLM 重复调用
  - 流式输出
